Convert part of svn repository to git

The repository directory to be checked out can be specified in the latter part of the repo URL.

git svn clone http://example.com/svn/project/folder
--no-minimize-url --no-metadata -A authors-transform.txt
folder.[1]

This clones the tdms directory in the svntom repo:

$ git svn clone file:///home/remarcil/svn/tomcat/trunk/tdms
Initialized empty Git repository in /usr/local/src/tdms/.git/
W: Ignoring error from SVN, path probably does not exist: (160013): Filesystem has no item: File not found: revision 100, path '/trunk/tdms'
W: Do not be alarmed at the above message git-svn is just searching aggressively for old history.
This may take a while on large repositories
Checked Ahrough projects/tracks/docs/notes/project_charter.txt
    A   projects/tracks/docs/notes/tracks_schedule.txt
    A   projects/tracks/docs/notes/blm_doc_subtypes.csv
    A   projects/tracks/docs/notes/security_layer.txt
...

"If you're trying to copy the entire history, and using git svn clone --stdlayout just use the URL that you would use to checkout the trunk with svn minus the /trunk portion at the end."[2]
